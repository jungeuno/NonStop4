[발표 내용 요약]
- oracle linux으로 자동화를 하기 위해서 oke로 변경을 하였다.
- docker hub대신에 oracle cloud container registry사용하고 있다.
- oke 환경으로 구성을 했고, 로드밸런스연결까지 하였고, 샘플 이미지까지 배포를 하였다.
- 오라클 클라우드 처음이라서 힘들었다.
- 파이프라인 뼈대 작성중
- 쿠버네티스 야물파일 작성중이다.

  - 네이밍 룰 제공
  - 개발자가 작성한 도커 파일 사용(우리 주제와 맞지 않음)
  - 사용자의 입력을 받아서 처리

- 전반적인 느낌이 짧은 기간 내에 모든 것을 개발하기 어렵고 완벽하게 개발하는 것이 쉽지 않기 때문에 과정이 중요하고, 이런 과정들을 결과 때 잘 발표했으면 좋겠다.결과물이 실행은 무조건 되어야 한다.

- IP관련 질문파드 개수가 제한되어있다는 약간 이상하다. -> 확인해보시겠다.

- replica를 조정해서 이중화를 진행할 때,워커 노드가 2개더라도, 파드가 각각으로 배포되는 것이 아니다.-> 어떻게 해결할 것이냐?

- 깃 허브에 왜 푸시를 하느냐? 재배포를 위함이다.

- 깃 허브 계정을 하드 코딩을 안하고 작업을 어떻게 해야할 지 고민해봐야 한다. 젠킨스에서도 깃 허브의 계정정보를 어떻게 가져와서 사용할 것인지도 고민해봐야 한다.

- 이중화가 있어서 궁금한 점 소스가 올리면 팓이 두개가 떠있는 건데 이중화 정책이 설정된 것이 있느냐?

- 이중화를 위해서 무엇을 진행하고 있는지? replicaset만 2로 설정하여서 사용하고 있느냐?active-active 관계인지?

- 프로메테우스랑 그라파나cpu랑 메모리가 0이라면 정상적인 서비스인가? 아닌가? 서비스의 현황이나, 기존의 트래픽 등을 고려해서 판단해야한다cpu사용량이 0인 서비스에 대해서 트래픽을 늘려보는 테스트를 진행해서 정상인지 비정상인지 구분할 수 있다.그런 정책들도 정하고 자동화를 해서 모니터링 부분에 포함을 했다면 더 좋은 기획이 되었을 것 같다!!!

- dns서버를 통해서 url을 발급받는 것을 자동화할 수 있는가? 할 수는 있는데 오라클은 잘 모르겠다.클라우드 벤더마다 도메인 발급하는 기능은 있을 것이다.dns로 발급받아서 로드밸런서 타고 접속을 할 수 있다.
